# Правила оформления отчётов в Typst

## Структура проекта

- Титульный лист — отдельный модуль (`title_page_prac.typ` или `title_page_lab.typ`), скопированный в локальную директорию из пресета
- Оформление листингов — отдельный модуль `listings.typ` (см. раздел «Листинги»)
- Основной файл отчёта — `report.typ` в локальной директории работы
- Расчётные и вспомогательные скрипты хранятся отдельно от Typst-файлов

## Страница

```typst
#set page(
  paper: "a4",
  margin: (top: 2cm, bottom: 2cm, left: 3cm, right: 1.5cm),
  numbering: none,       // глобально выключена — включается после TOC
  number-align: center,
)
```

## Шрифт и абзац

- Основной шрифт: **Times New Roman, 14pt**
- Межстрочный интервал: **1.5** (`leading: 0.65em`, `spacing: 1.5em`)
- Красная строка: `first-line-indent: 1.25cm`
- У заголовков красная строка сбрасывается: `set par(first-line-indent: 0pt)`
- Все заголовки, подписи и основной текст оформляются тем же шрифтом и кеглем, если преподаватель не требует иного

```typst
#set text(font: "Times New Roman", size: 14pt, lang: "ru")
#set par(leading: 0.65em, spacing: 1.5em, first-line-indent: 1.25cm)
```

## Нумерация страниц

- Титульный лист и страница содержания **не нумеруются** (глобально `numbering: none`)
- Нумерация включается после `#pagebreak()` перед первым разделом
- Счётчик страниц устанавливается вручную в **3** (физический номер первой нумеруемой страницы)

```typst
#set page(numbering: "1")
#counter(page).update(3)
```

## Структура документа

1. Титульный лист (`#title_page(...)`) → `#pagebreak()`
2. Содержание (`#outline(...)`) → `#pagebreak()`
3. Включение нумерации страниц (`#counter(page).update(3)`)
4. Введение
5. Разделы по заданиям (= Задание N. …)
6. Заключение

### Содержание

- Заголовок «СОДЕРЖАНИЕ» — по центру, жирный, 14pt
- Если заголовок содержания выводится вручную, у `outline` следует отключить встроенный заголовок: `#outline(title: none, ...)`
- Записи первого уровня — жирные
- Выравнивание — по левому краю (`indent: auto`)

## Заголовки

- Все уровни: Times New Roman, 14pt, **жирный**
- Отступ до и после заголовка задаётся через `block(above:, below:)`, а не через ручные `v(...)`
- Обычно отступ до и после заголовка соответствует межабзацному интервалу основного текста

## Листинги

### Модуль `listings.typ`

Всё оформление листингов выносится в отдельный файл. Экспортирует три символа:

| Имя | Тип | Назначение |
|---|---|---|
| `setup-listings` | функция-обёртка | применяется через `#show: setup-listings`; переопределяет оформление всех блочных `raw` |
| `listing` | функция | подпись + тело листинга |
| `tbl-caption` | функция | подпись таблицы |

Подключение в `report.typ`:
```typst
#import "listings.typ": setup-listings, listing, tbl-caption
#show: setup-listings
```

### Оформление блока кода

- Шрифт листинга задаётся явно и может отличаться от основного текста только если это согласовано требованиями
- Если специальных требований нет, допускается оформление листингов тем же гарнитурным стилем, но меньшим кеглем
- Фон: `luma(245)` (светло-серый)
- Левая граница: `2pt + luma(180)` (акцентная полоса)
- Внутренние отступы: `(x: 10pt, y: 8pt)`
- Скругление углов: `3pt`
- **Нумерация строк**: отдельная левая колонка, уменьшенный размер, приглушённый цвет

### Подпись листинга

- Располагается **над** блоком кода
- Выравнивание: **по левому краю**
- Формат: `**Листинг N.M** — Название`
- Нет красной строки (`first-line-indent: 0pt`)
- Подпись и сам блок листинга оформляются как отдельные блоки с явно заданными `above`/`below`
- Между подписью и листингом делается небольшой специальный отступ, как правило меньше обычного межабзацного

```typst
#listing("1.2", "Функция clean_text_v2")[
  ```python
  ...
  ```
]
```

### Содержимое листингов

- В листинги включается **только необходимый** для понимания код, не весь модуль
- **Комментарии в коде недопустимы** — пояснения выносятся в основной текст отчёта
- Нумерация сквозная внутри задания: `1.1`, `1.2`, …, `2.1`, `2.2`, …

## Таблицы

- Подпись — **над** таблицей, выровнена по левому краю
- Формат: `**Таблица N.M** — Название`
- Сама таблица может быть выровнена по центру страницы независимо от подписи
- Заголовочная строка: заливка `luma(220)`, остальные строки — белые
- Обводка: `0.5pt`
- Внутренние отступы: `6pt`
- Нет красной строки перед/после (`first-line-indent: 0pt` при вызове)
- Подпись и таблицу следует оформлять как отдельные блоки с явно контролируемыми `above`/`below`

```typst
#tbl-caption("1.1", "Примеры заголовков до и после очистки")
#align(center)[
  #table(
    columns: (...),
    fill: (_, row) => if row == 0 { luma(220) } else { white },
    stroke: 0.5pt,
    inset: 6pt,
    ...
  )
]
```

## Рисунки

- Рисунок располагается отдельным блочным элементом; при необходимости выравнивается по центру страницы
- Подпись — **под** рисунком (либо над — по ситуации), по центру, жирный
- Формат: `Рисунок N.M — Название`
- Подпись должна быть полноширинным блоком, внутри которого текст центрируется
- Рисунок и подпись оформляются отдельными блоками; между ними используется уменьшенный специальный отступ, а перед рисунком — отдельный отступ от основного текста

## Импорты и модульность

- Пресет титульного листа **копируется** в локальную директорию работы (не импортируется по абсолютному пути)
- `listings.typ` — переиспользуемый модуль, не зависит от конкретной работы
- Правило `show raw` **не дублируется** в `report.typ` — оно живёт только в `listings.typ`

## Формулы и единицы измерения

- Обозначения в формулах выбираются в соответствии с языком отчёта; если документ на русском, буквенные индексы также предпочтительны на русском
- Единицы измерения, знаки умножения, степени и приближённые равенства в математическом режиме записываются средствами Typst, а не Unicode-символами
- Предпочтительны записи вида:

```typst
$ "1,2" dot 10^(-3) "Па" dot "с" $
$ "789" "кг"/"м"^3 $
$ "6240" "с" approx "1,73" "ч" $
```

- Следует избегать прямой вставки символов вроде `·`, `²`, `³`, `≈`, если выражение уже набирается в математическом режиме

## Управление отступами у объектов

- Не полагаться только на `par.spacing` для рисунков, таблиц, листингов, `align`, `raw`, `table` и других блочных элементов
- Для точного управления расстояниями вокруг таких объектов использовать `block(above:, below:)`
- Ручные `v(...)` применять только как исключение, когда блочной модели недостаточно
- Подпись и сам объект лучше оформлять раздельно: это даёт независимый контроль над выравниванием и вертикальными интервалами

## Типичная структура раздела по заданию

```
= Задание N. Название

== Постановка задачи
(краткое описание что нужно сделать)

== Подраздел (метод / алгоритм / функция)
(описание подхода в тексте)

#listing("N.M", "Название листинга")[
  ```python
  ...
  ```
]

== Результаты
(анализ результатов в тексте)

#tbl-caption(...)
#table(...)
```
